# Change Budget Controller v1.0

## 1. 目的

変更可能な量の上限を定め、良い記事の破壊を防ぐ。

## 2. 基本予算

| Preservation Score | 基本Change Budget |
|---|---:|
| 90〜100 | 0〜5% |
| 75〜89 | 5〜15% |
| 60〜74 | 15〜30% |
| 40〜59 | 30〜60% |
| 0〜39 | 60〜100% |

## 3. 補正

### 予算を減らす

- S/Aランク：-5〜-10%
- 高CTR：-5%
- 改善後28日未満：-10%
- 過去改善で成果上昇：-10%
- 独自体験が多い：-5%

### 予算を増やす

- critical/high誤情報：必要範囲まで増加
- INTENT_MISMATCH：+10〜30%
- REBUILD_CANDIDATE：+20〜40%
- Coverage 40未満：+20%
- 一貫性high問題が複数：+15%

## 4. Change Budgetの定義

文字数比率だけではない。

次を総合する。

- 変更段落数
- 変更見出し数
- タイトル変更
- FAQ変更
- 内部リンク変更
- 削除量
- 追加量
- 構造変更量

## 5. Budget超過

改善案が予算を超える場合：

1. 変更対象を優先順位付け
2. 不要変更を削除
3. 局所修正へ縮小
4. それでも必要ならriskを1段階上げる
5. Quality Gateで再審査

## 6. 例外

次はBudget制約より優先する。

- 明確な誤情報
- 危険情報
- 法的・医療・金融上の重大問題
- 読者を誤認させる価格・条件
- 現在利用不能な手順

## 7. 禁止事項

- 文体統一のために記事全体を書き換える。
- AIらしい表現へ揃える目的で予算を消費する。
- 良い具体例を短文化のために削除する。
- 変更率を文字数だけで算出する。
